\# Portfolio QA Automation - Othman KHADROUF



Bienvenue sur mon dépôt de démonstration technique.

Ce projet a été conçu pour illustrer mes compétences en automatisation de tests dans un contexte \*\*E-commerce / B2B\*\*.



\## Objectifs du POC

Tout simplement démontrer 3 compétences clés pour un QA moderne :

1\. \*\*Tests E2E (End-to-End) :\*\* Simulation d'un parcours d'achat complet.

2\. \*\*Data Driven Testing :\*\* Utilisation de jeux de données dynamiques pour valider plusieurs profils utilisateurs.

3\. \*\*API Testing :\*\* Validation directe des services backend (sans interface) pour garantir la rapidité et la fiabilité.



\## Stack Technique

\* \*\*Framework :\*\* Playwright (TypeScript)

\* \*\*Architecture :\*\* Tests isolés et parallélisés

\* \*\*CI/CD :\*\* GitHub Actions (Tests lancés automatiquement à chaque push)

\* \*\*BDD :\*\* Exemples de fichiers Gherkin dans le dossier `/specifications-metier`



\## Comment lancer les tests

Si vous souhaitez exécuter ce projet en local :
